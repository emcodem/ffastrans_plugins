Files Find Processor
This Processor finds files and folders and outputs a simple JSON array with the found items.

Note that you have to configure the "Path" where to start finding files!
Use %s_original_path% to search files in the same directory as the current source file of this job resides.

Usage examples :
Populate Variables Processor: use $readarray('%s_file_list%', 2)
Delete Files Processor: Provide a user_variable in the "File Variable" output and use the same variable in the input of the Delete Files Processor.

Path
Base path, this will not be included in output list

Include and Exclude patterns
Use like searching files in Windows Explorer, you can provide a comma separated list, e.g. *.m2v,*.wav

Older Than
Filter Files and Folders by age

Recurse:
Recurse into subdirectories

Skip Folders/Files
Do not include Folders/Files in output list

File Variable
If you provide a user_variable here, like %s_file_list%, it will be populated with the JSON array of found files

Count Variable
If you provide a user_variable here, you get the count of found items

Single File Outputs
If you provide a user_variable, it will be populated with the X'th item of the found list. You don't need to use $readarray function in follow-up processors.
