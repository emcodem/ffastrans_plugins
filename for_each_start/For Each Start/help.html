This processor node will create one Branch for each item in a List (JSON Array) of items.
In other words, anything connected to the right of this processor will be executed as often as you have items in your input list.

Foreach List
A JSON array of Items. Most simple example: ["0","1"]
In Production, the List of items might be generated by another processor, e.g. the Files Find Processor.

Target Value Variable

This variable will contain the current Value from the list. Most likely you want to use %s_source% here if your foreach list consists of files.

Target Count Variable:

This variable will contain the current Index of the current list item. Rarely used.

Hide main branch:

The Branch that is incoming to this Foreach Processor actually ends here. You can decide if you want to see it in the job monitor or not.

Important Notes:

Due to how this processor works, a Hold-Synchronize Node later in the workflow cannot work correctly as it is not able to calculate the number of created branches anymore.
Also, you can only connect one processor on Input and one processor on output of this Node.
This way, the foreach processor is enabled to give you error messages in case your workflow does something that is not supported by foreach.
